Model Fusion Proxy 架构总览

基于 MoE 的智能模型代理网关 — 本地 + 云端混合路由

路由策略

  • • 8 种意图分类（代码 / 推理 / 创意 / 中文语境 / 视觉 / 工具调用 / 通用）
  • • 每类：1 个首选 + 4 个后备模型
  • • 通用 & 创意 → 本地 Gemma 2B 优先（零成本）
  • • 代码 & 工具调用 → DeepSeek V4 Pro 首选
  • • 推理 → Gemini 2.5 Pro 首选
  • • 中文语境 → GLM 5.2 首选

MoA 融合（显式开启）

  • • 仅通过 model="model-fusion" 显式触发
  • • 阶段 1：3 个云端 Panel 并行草拟（≤60s）
  • • 阶段 2：Judge（Gemini 2.5 Pro）合成草案
  • • 阶段 3：2 个批评家 + 本地 AST 校验
  • • 阶段 4：Judge 精修输出终稿
  • • 策略：hybrid（本地 Qwen 9B 为 Panel + 云端 Judge）
  • • 空响应熔断：自动回退 Fallback 快速通道

容错与安全

  • • P0 修复：删除 "claude" 子串融合触发 — 融合改为显式开启
  • • 非流式：并发竞速（先到先赢，失败者 cancel）
  • • 流式：串行后备（无法中途切换模型）
  • • 本地 GPU 信号量：最多 2 并发、不重试（防 OOM）
  • • 全局并发信号量：64 并发请求上限
  • • 空工具清洗：进入上游前剥离无名函数
  • • .env 缓存：启动时解析一次（避免每次请求读盘）
